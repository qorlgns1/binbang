# Binbang Beta Onboarding Guide (W4-D5-T1)

목표:
- 베타 사용자 10명이 실제로 알림 1개 이상 등록
- 첫 48시간 내 알림/클릭아웃 데이터 수집

## 1) 모집 기준

- [ ] 신혼여행/휴양지 숙소를 직접 예약하려는 사용자
- [ ] 체크인 날짜가 확정되어 있고 2주 이상 남은 사용자 우선
- [ ] 이메일 수신 동의 가능 사용자

권장 세그먼트(각 3~4명):
- 몰디브
- 발리
- 칸쿤

## 2) 온보딩 전 준비

- [ ] 알림 등록 경로 점검: `/accommodations/new`
- [ ] 대시보드 점검: `/dashboard`
- [ ] 수신거부 링크 정상 동작 점검
- [ ] 운영자 모니터링 경로 점검: `/admin/ops`

## 3) 사용자 온보딩 플로우 (1인당 5분)

1. 로그인
2. 호텔 검색(호텔명/도시)
3. 체크인/체크아웃/인원 입력
4. 수신 동의 체크 후 알림 등록
5. 대시보드에서 등록 상태 확인

## 4) 운영자 체크리스트

- [ ] 온보딩 직후 `agoda_consent_logs`에 `opt_in` 기록 확인
- [ ] `agoda_poll_runs` 생성 확인
- [ ] `agoda_notifications` 상태 추적 (`queued/sent/failed/suppressed`)
- [ ] 클릭아웃 이벤트(`type = clickout`) 집계 확인

## 5) 사용자 안내 문구 템플릿

```
[Binbang 베타 안내]
등록하신 호텔의 가격 하락/가용성 변동을 감지하면 메일로 알려드립니다.
알림은 실시간 재고 변동에 영향을 받을 수 있으며, 최종 예약 가능 여부는 Agoda 페이지에서 확인해주세요.
```

## 6) Day-1 운영 보고 항목

- 온보딩 완료 인원 수
- 알림 등록 수 / 활성 알림 수
- 이메일 발송 성공률 (sent / (sent + failed))
- 오탐 후보 수 (verify reject + failed/suppressed 케이스)
- 클릭아웃 수

리포트 실행:
- `reports/daily.sql`

## 7) 베타 종료 기준

- [ ] 사용자 10명 모두 알림 1건 이상 등록
- [ ] 최소 1건 이상의 이메일 발송 성공
- [ ] 최소 1건 이상의 클릭아웃 발생
- [ ] 주요 이슈(오탐/지연/미발송)와 개선 백로그 정리 완료
